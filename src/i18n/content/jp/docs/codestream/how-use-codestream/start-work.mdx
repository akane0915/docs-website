ちょっとしたバグ修正や大きな新機能など、新しい作業に取り掛かるには、多くの手順やさまざまなサービスが必要です。課題追跡サービスにアクセスして、自分に割り当てられたすべての課題を見つけ、作業する課題を選び、そのステータスを更新します。次に、ターミナルまたはGitHubに移動して、機能ブランチを作成します。そして最後に、Slackでステータスを更新して、チームメイトに作業内容を知らせます。New Relic CodeStreamを使えば、IDEからワンステップでこれらの作業を行うことができます。

はじめに、CodeStreamペインの **Issues** セクションにアクセスし、チームの課題追跡サービスに接続します。

![課題サービスへの接続方法を示すスクリーンショット](/images/IssuesSection-Connect.png "課題サービス")

接続すると、自分に割り当てられたすべての課題を、プロジェクト、ボード、リストなどでフィルタリングし、自分の好みに合わせてカスタマイズして見ることができます。複数のサービスに接続することも可能です。

![課題リストのスクリーンショット](/images/IssuesSection-Backlog.png "課題リスト")

利用している課題追跡サービスでは、どの課題も見ることができます。

![ウェブ上での課題の閲覧方法を示すスクリーンショット](/images/WorkingOn-OpenWeb.png "オープンイシュー")

課題をクリックすると、その課題への取り組みを開始できます。

![課題への取り組みを開始するためのスクリーンショット](/images/StartWork2.png "スタートワーク")

あなたのチームが機能ブランチのGitワークフローを使用している場合、作業を行うためのブランチを作成することができます。ドロップダウンを使って、ベースブランチを変更するか、機能ブランチの名前を編集します。

![ブランチ名の編集画面です。](/images/StartWork-BranchOptions.png "ブランチオプション"))

管理者であれば、組織のためにブランチの命名テンプレートを設定して、機能ブランチが一貫した名前になるようにすることができます。

![ブランチのテンプレートを示すスクリーンショット](/images/StartWork-BranchTemplate2.png "枝のテンプレート")

使用している課題追跡サービスに応じて、課題のステータスを更新することもできます。例えば、Trelloを使用している場合、選択したカードを別のリスト（例えば、 **In Progress** ）に移動させることができます。

もし、課題に関連していない作業を行う必要がある場合は、 **Start ad-hoc Work** をクリックして、課題なしで作業を開始するか、 **New issue** をクリックして、CodeStreamから直接課題追跡サービスに課題を作成することができます。また、課題をエディタ上の特定のコードブロックに関連付けることもできます。
