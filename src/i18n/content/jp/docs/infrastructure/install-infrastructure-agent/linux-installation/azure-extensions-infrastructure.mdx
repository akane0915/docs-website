当社の [インフラストラクチャ](/docs/welcome-new-relic-infrastructure) エージェントは、 [Microsoft Azure の拡張機能](https://docs.microsoft.com/en-us/azure/virtual-machines/extensions/overview) を使ってインストールできます。

<CollapserGroup>
  <Collapser
    id="what-is-extension"
    title="Azureエクステンションとは？"
  >
    Azure仮想マシン(VM)エクステンションは、Azure VMのデプロイ後の設定や自動化タスクを提供する小さなアプリケーションです。たとえば、仮想マシンがソフトウェアのインストール、ウイルス対策、スクリプトの実行を必要とする場合、VMエクステンションを使用することができます。Azure CLI、PowerShell、Azure Resource Managerテンプレート、およびAzureポータルは、すべてAzure VMエクステンションを実行できます。エクステンションは、新しいVMの展開にバンドルすることも、既存のシステムに対して実行することもできます。
  </Collapser>
</CollapserGroup>

New Relic ではこれらの拡張機能を使用して、Infrastructure エージェントのインストールを容易にしています。これにより、エージェントを手動で更新する必要がなくなり、さらに重要なことに、 [Azure スケールセット](https://docs.microsoft.com/en-us/azure/virtual-machine-scale-sets/overview) に合わせてエージェントを自動的にスケールさせることができます。

## Azureの拡張機能を使ってLinux用のインフラをインストール [#install-infra-azure-extensions]

Azure 拡張機能を使ってエージェントをインストールするには、次のようにします。

1. 任意のAzure VM、Azureスケールセット、またはスケールセットを使用するAzure Service Fabricに移動し、 **Extensions** タブを選択します。

   ![azure_ext01.png](./images/azure_ext01.png "azure_ext01.png")

2. **New Relic Infrastructure** のエクステンションを選択します。

   ![azure_ext02.png](./images/azure_ext02.png "azure_ext02.png")

3. 指示に従って、 [New Relic のライセンスキーを追加してください。](/docs/accounts/accounts-billing/account-setup/new-relic-license-key)

   ![azure_ext03.png](./images/azure_ext03.png "azure_ext03.png")

4. これでいいんだ！」と。
